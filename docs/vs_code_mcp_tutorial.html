7Start the Database

What is a database? Think of it as a digital filing cabinet for all your writing data - characters, plot points, series information, etc. We use PostgreSQL running in Docker so you don't need to install database software directly.
IMPORTANT: Check Docker Desktop First!
  • Look for the Docker whale icon in your system tray (Windows) or menu bar (Mac)
  • If not visible, open Docker Desktop and wait for it to start (can take 1-2 minutes)
  • Wait for "Docker Desktop is running" status

Start your PostgreSQL database using VS Code terminal:

Start database (Windows PowerShell) VS Code Terminal
.\scripts\start-database.ps1
This PowerShell script:
  • Checks if Docker Desktop is running
  • Starts your PostgreSQL database container
  • Sets up proper database configuration
  • Provides helpful status messages
Start database (Mac/Linux) VS Code Terminal
docker-compose up -d
This command:
  • Starts services defined in docker-compose.yml
  • Runs in background (-d = detached mode)
  • Sets up PostgreSQL database for the project

What Happens in Your VS Code Terminal

Database is healthy and ready!
PS C:\...\MCP-tutorial>

Verify database is running:

Check running containers VS Code Terminal

Success indicators: You should see a container with "postgres" in the name, and status showing "Up" or "healthy".
  • Docker Desktop was running before command

8Create Database Schema

What is a database schema? It's the blueprint that defines what tables (like spreadsheet tabs) and columns (like spreadsheet headers) your database has. A migration script builds these tables automatically.

Run the migration to create your database structure:

Run core schema migration VS Code Terminal
node src/shared/run-migration.js 001_create_core_schema.sql

Explore the Migration File in VS Code

While the migration runs, you can explore what it's doing:

  • In Explorer: Navigate to migrations ‚Üí 001_create_core_schema.sql
  • Click the file to open it and see the SQL code
  • Notice: It creates tables for authors, series, books, timeline, metadata
What the migration creates: Tables for organizing your writing projects, with proper relationships and indexes for good performance.
Common issues:
  • "Database connection failed": Ensure database is running (check with docker ps)
  • "Migration already applied": This is fine! Means tables already exist
  • "Permission denied": Check your .env file settings
  • Migration completed without errors
  • Saw success message in terminal
  • Can view migration file in VS Code

10Generate Claude Desktop Configuration

What is this configuration? Claude Desktop needs to know where your MCP tools are located so it can use them. Think of it like installing browser extensions - you need to tell the browser which ones to load.

Generate the configuration file for Claude Desktop:

Generate Claude Desktop configuration VS Code Terminal
.\scripts\generate-configs.ps1 -Claude

Watch the Configuration File Appear

After running the script, you'll see a new file in VS Code:

  • claude-desktop.json appears in your Explorer
  • Click on it to open and view the configuration
  • See the MCP servers it configures (author-server, series-server, etc.)
Configuration File Contents (preview):
{
  "mcpServers": {
    "author-server": { ... },
    "series-server": { ... }
  }
}

Manual Installation Required

The script generates the file, but you need to install it in Claude Desktop's configuration folder:

Using VS Code for Windows:

  1. In VS Code Explorer: Right-click "claude-desktop.json" ‚Üí "Copy"
  2. Open File Explorer: Press Win + E
  3. Navigate to config folder: Type %APPDATA%\Claude in address bar, press Enter
  4. Paste and rename: Right-click ‚Üí Paste, then rename to "claude_desktop_config.json"
Alternative instructions:
  1. File > Help > Troubleshooting > Enable Developer Mode
  2. File > Developer > Open App Config > File...

Using VS Code for Mac:

  1. In VS Code Explorer: Right-click "claude-desktop.json" ‚Üí "Copy"
  2. Open Finder: Click Finder icon
  3. Navigate to config folder: Press ‚åò + Shift + G, type ~/Library/Application Support/Claude
  4. Paste and rename: Right-click ‚Üí Paste, then rename to "claude_desktop_config.json"
CRITICAL: Restart Claude Desktop
After copying the config file:
  1. Completely exit Claude Desktop (not minimize or X - actually quit) File -> Exit
  2. Wait 5 seconds
  3. Open Claude Desktop again
  4. Sign in if prompted
  • claude-desktop.json file appears in VS Code
  • Can view configuration file contents in VS Code
  • Configuration copied to Claude Desktop folder
  • File renamed to "claude_desktop_config.json"
  • Claude Desktop completely restarted
